# Post Frontmatter Template

**File naming convention:** `YYYY-MM-DD-post-slug.md` (e.g., `2026-02-11-building-my-learnbase-mvp.md`)

The date prefix makes it easy to find files chronologically. The slug in frontmatter controls the URL, not the filename.

Copy this to start a new post:

```toml
+++
title = "Post Title"
slug = "post-title"           # Determines URL, never change once published
date = 2026-01-01
draft = false

[taxonomies]
tags = ["tag1", "tag2"]
series = ["series-name"]      # Optional, for multi-part content

[extra]
series_order = 1              # Optional, position in series
+++
```

## Field Reference

| Field | Required | Description |
|-------|----------|-------------|
| title | Yes | Post title (can change without breaking links) |
| slug | Yes | URL path â€” never change once published |
| date | Yes | Publication date (YYYY-MM-DD) |
| draft | No | Set `true` to hide from listings |
| updated | No | Last updated date |

## Taxonomies

| Taxonomy | Description |
|----------|-------------|
| tags | Topic tags (e.g., "rust", "zola", "web") |
| series | Multi-part post grouping |

## Extra (optional)

| Option | Description |
|--------|-------------|
| series_order | Position within a series (1, 2, 3...) |

## Post Body Structure

After the frontmatter, posts use a two-part format:

```markdown
## Reflections

<!-- TODO: Write reflections/commentary section -->
<!-- This section is ALWAYS written by the human author, never generated by AI -->

---

## Tutorial: [Topic]

[Detailed reproducible steps here]
```

The **Reflections** section should be left blank (with placeholder comment) when AI is generating the post. The human author fills it in separately.
